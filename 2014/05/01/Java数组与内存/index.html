Java数组初始化

Java数组初始化分

  1. 静态初始化：初始化时有程序员显示指定每个数组元素的初始值，由系统决定数组的长度。

  2. 动态的初始化：初始化时程序员只指定数组长度，有系统为数组元素分配初始值。

    这两种初始化不能并存，也就是说即指定数组的长度，又分配每个元素的值。

    一旦数组初始化后，其内存空间分配即结束，后面只能改变其元素的值，而不能改变其长度。

    定义数组的时候不一定要进行初始化，因为Java的数组是引用类型的变化，它并不是数组对象本身，只要让输入变量指向有效数组对象即可使用。

    所有局部变量都是放在栈内存里面保存的，不管其是基本类型的变量，还是引用类型的变量，都是存储在各自的方法栈区中，但是引用类型变量所引用的对象则总是存储在堆内存中。
