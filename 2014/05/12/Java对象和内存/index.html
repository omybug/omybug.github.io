实例变量和类变量

实例变量和类变量：

  • 实例变量：没有使用static修饰的成员变量是实例变量。
  • 类变量：使用static修饰的成员变量是类变量。
    在同一个JVM内，每个类只对应一个Class对象，但是每个类可以创建多个实例，所以可有有多个块实例内存，但是只有一块内存存放类。同时类变量的初始化时机比实例变量初始化要早。

父类构造器

在初始化子类的时候，系统首先会初始化父类的非静态初始化块、构造器，然后在执行子类的非静态初始化块和构造器。
子类调用父类构造器分一下几种：

  1. super
  2. this
  3. 没有super也没有this，直接调用父类无参构造器。
    需要注意，super和this只能写在构造第一行代码，所以，super和this在构造器中只会出现其中之一，而且也只能调用一次。

当变量的编译是类型和运行时类型不同时，通过该变量访问它引用的对象的实例变量时，该实例变量的值由声明该变量的类型决定。但通过该变量调用它引用的对象的实例方法时，该方法行为将由它实际所引用的对象来决定。因此，访问this.age的时候，实际访问的是Person.age；执行this.display的时候，执行的是Student.display。

内存中的实例

通过上面这个例子，可以看出虽然B继承自A，但是在内存中只存在一个B对象，并不存在A对象，也就是为什么子类不同使用return super，但是可以使用return this，因为并不存在super这个对象。
当程序创建一个子类对象时，系统不仅会为该类中定义的实例变量分配内存，也会为其父类中定义的所有实例变量分配内存，即使子类定义了与父类中同名的实例变量，所以出现了super这个词，来区分子类实例变量和方法与父类之间的调用。

final修饰符

final实例变量初始化只能在

1.定时final实力变量时指定初始值。
2.在非静态初始化块中为final实例变量指定初始值。
3.在构造器中为final实例变量指定初始值。

final类变量只能在

1.定时final类变量时指定初始值；
2.在静态初始化块中为final类变量指定初始值。
final类变量不能在构造函数可非静态初始化块中指定初始值。
此外，final变量和其他变量不一样，其他变量都是在代码执行或者构造函数里初始化，但是final在编译时就直接把具体的值当变量使用。所以下面代码的结果是：17.
